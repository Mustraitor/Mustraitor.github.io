操作系统

操作系统

计算机系统体系结构

  • 冯·诺依曼结构

单处理系统

  • 只有一个CPU，执行通用指令集
  • 带有专用处理器，专用处理器执行有限的指令集，而并不执行用户进程

多处理器系统

  • 非对称处理器：每个处理器有其特定任务，一个主处理器 + 其它处理器，主处理器调度从处理器，并安排任务

  • 对称处理器：每个处理器都参与完成操作系统的所有任务，无主从关系

  • 优点：

    增加吞吐量

    规模经济

    增加可靠性

操作系统结构

多道程序设计

  • 内存中同时装入多道程序

  • 通过安排作业，让CPU始终有一个作业执行，提高CPU利用率

  • 作业首先保存在磁盘的作业池（job pool）

分时系统

  • 通过切换作业来执行多个作业，切换频率快（伪并行）

操作系统的执行

  • 中断驱动的，事件是由中断或陷阱引起的
  • 陷阱（中断/异常）是一种软件生成的中断，或源于出错，或源于用户程序的特定请求

双重模式

  • 用户模式（user mode）和内核模式 （kernel mode）
  • 硬件通过模式位来表示当前模式，用户模式（1），内核模式（0）
  • 内核模式才允许执行特权指令
  • 当计算机系统执行用户应用时，系统处于用户模式，当用户应用通过系统调用，请求操作系统服务时，系统必须从用户模式切换到内核模式

  • 双重模式执行提供保护手段，以便防止操作系统和用户程序受到错误用户程序的影响

定时器

  • 维持控制CPU，防止用户程序陷入死循环，或不调用系统服务并且不将控制返给操作系统

进程管理

  • 进程：执行中的程序
  • 程序本身不是进程，程序是被动实体，而进程是主动实体
  • 进程是系统的工作单元，所有进程是并发执行的

内存管理

  • 一个大的字节数组
  • 内存保留多个程序（多道程序）

操作系统在内存管理中的作用

  • 分配内存
  • 回收内存
  • 虚拟内存（Virtual Memory）提供抽象
  • 页表管理
  • 页式内存管理和缺页异常处理
  • 内存共享

存储管理

  • 提供信息存储的统一逻辑视图
  • 逻辑单元，即文件

高速缓存

计算环境

  • 传统计算
  • 移动计算
  • 对等计算
  • C/S
  • 虚拟化
  • 云计算

2 操作系统服务

  • 提供环境，以便执行程序

  • 用户界面：命令行界面（Shell）、批处理界面、图形用户界面（GUI）
  • 程序执行
  • I/O操作
  • 文件系统操作
  • 通信：一个进程与另一个进程交换信息（内存共享、消息交换）
  • 错误检测
  • 资源分配
  • 记账
  • 保护与安全

系统调用

  • 提供操作系统服务接口

六大类型

  • 进程控制
    多个进程会共享数据
    一个进程锁定，其它进程不能访问该数据
  • 文件管理
  • 设备管理
  • 信息维护
  • 通信
  • 保护

操作系统结构

  • 简单结构

  • 分层法

  • 微内核：对内核进行模块化
    为客户端程序和运行在用户空间中的各个服务提供通信
    优点：内核更小、便于扩展操作系统、更好的安全性与可靠性、便于移植。

    缺点：增加系统开销、微内核的性能会受损。

  • 模块：采用可加载的内核模块

  • 混合系统（如Android分层模式）

系统引导

  • 加载内核以启动计算机的过程
  • 引导程序存储在只读存储器（ROM）

3 进程

  • 执行中的程序实体, 还包含当前活动，数据结构
  • 进行资源分配和调度的基本单元
  • CPU密集型进程和I/O密集型进程
  • 具有独立的地址空间

进程状态

  • 新的（new）
  • 运行（running）
  • 等待（waiting）
  • 就绪（ready）
  • 终止（terminated）

僵尸进程：是已经 终止执行，但它的 PCB（进程控制块）还存在，等待父进程读取其退出状态的进程。（死了但还没收尸的进程）

孤儿进程：是 父进程先退出，而子进程仍在运行 的进程

进程控制块（PCB）

  • 记录进程的相关信息（仓库）
    进程状态、程序计数器、CPU调度信息、内存管理信息、记账信息、I/O状态信息

进程调度

  • 作业队列：所有进程
  • 就绪队列（链表）：驻留在内存中、就绪的、等待的
  • 设备队列：I/O设备的进程

调度程序

  • 长期调度程序（作业池选择进程，加到内存）
  • 中期调度程序（内存中移除再进入）
  • 短期调度程序（就绪状态到运行状态）

上下文切换

  • 让 CPU 从一个进程切换到另一个进程的场景

  • 保存当前进程状态

  • 恢复另一个进程状态

进程运行

进程创建

  • 进程标识符（pid）

进程终止

进程通信

  • 消息传递
  • 共享内存

客户机/服务器通信

  • 套接字
  • 远程过程调用（RPC）
  • 管道（普通管道、命名管道）

4 多线程编程

  • 线程是CPU使用的一个基本单元
  • 并行：同时可以执行多个任务（多核）
  • 并发：单核系统上，通过CPU上下文切换，实现同时执行多个任务

多线程模型

  • 用户线程
  • 内核线程
  • 多对一模型
  • 一对一模型
  • 多对多模型
  • 双层模型

线程池

  • 在进程开始之前，创建一定数量的线程，加到池中等待工作

5 进程调度

先到先服务调度（First Come First service, FCFS）

  • 非抢占的
  • FIFO队列实现

最短作业优先（Shortest Job First, SJF）

  • 先执行时间短的进程
  • 平均等待时间最短
  • 用于长期调度

​ 非抢占SJF调度：会等待进程执行完，再从剩余进程中选取最短

​ 抢占SJF调度：执行进程的剩余时间会与到达进程执行时间进行比较

优先级调度

  • 按最高优先级分配CPU

  • 相同优先级的进程按FCFS顺序调度

  • 主要问题：无穷阻塞/饥饿
  • 解决方案：老化，逐渐增加在系统中等待很长时间的进程的优先级

轮转调度（Round-Robin, RR）

  • 为每个进程分配不超过一个时间片的CPU执行
  • CPU按顺序访问就绪队列中的进程

  • 性能取决于时间片大小
    时间片小，会频繁上下文切换
    时间片大，类似FCFS

多级队列调度

  • 将就绪队列分成多个单独队列

多级反馈队列调度

  • .进程在队列之间迁移

多处理器调度

方法

  • 非对称多处理
  • 对称多处理

处理器的亲和性

  • 软亲和
  • 硬亲和

负载平衡

  • 推迁移
  • 拉迁移

实时CPU调度

  • 软实时系统
  • 硬实时系统（严格截止时间）

最小化延迟

  • 中断延迟
  • 调度延迟

优先权调度

  • 支持抢占的基于优先权的算法（保证软实时）
  • 特性：进程是周期性的，0 <= t <= d <= p（保证硬实时）

单调速率调度

  • 抢占的、静态优先级
  • 周期越短，优先级越高
  • 调度N个进程的最坏情况下CPU利用率：N（2^1/n - 1）

最早截止期限优先调度（EDF）

  • 截止期限越早，优先级越高

比例分享调度

8 内存管理策略

内存：很大的字节数组

基地址寄存器（base register）：含有最小的合法的物理内存地址

界限地址寄存器(limit register)：指定了范围大小

物理地址 = 基地址寄存器的值 + 逻辑地址

交换：当内存不够时，把暂时不用的整个进程从内存移到外存；等需要时再换回来。

标准交换：进程在内存与备份存储之间移动

地址绑定

  • 编译时：文件转二进制文件

  • 加载时：加载到内存

  • 执行时：cpu读取指令

按字节编址：每个地址对应 1 个字节的内容

地址 存的内容
0x1000 0xAB
0x1001 0x34
0x1002 0xF0

程序的链接：

  • 静态链接
  • 装入时动态链接
  • 运行时动态链接

程序的装入：

加载到内存中

  • 绝对装入
  • 静态重定位
  • 动态重定位

连续内存分配管理

**连续内存分配：**进程的所有数据必须在物理内存里连成一条整块

  • 单一连续分配
  • 固定分区分配（分区大小不相等）
  • 动态分区分配

动态存储分配问题（动态分区分配）：在运行过程中，根据进程的实际大小动态划分内存，不预先分好固定大小的块。

常用方法：

  • 首次适应：找到 第一个大小足够的 空闲块
  • 最优适应：最小的、但又能装下进程大小的空闲块
  • 最差适应：最大的空闲块 分给进程

内存碎片

  • 外部碎片：多个小、分散的空闲区，不能合并
  • 内部碎片：分配块内部没用完的空间

内存回收：把这块空间标记为空闲，并与相邻空闲块合并，避免外部碎片增加

伙伴算法：相邻的空闲块合并（类似2048）

离散内存分配管理

分段：是按程序逻辑划分内存的方式，每个段是逻辑上独立的单元

  • 单位：逻辑段（代码段、数据段、栈段、堆段等）
  • 地址形式：逻辑地址 = <段号, 段内偏移>
  • 物理地址计算：物理地址 = 段基址 + 段内偏移
  • 段表：记录每段的基址和长度（段长），用于地址映射和保护

分段的特点

  1. 逻辑性

    • 按程序结构划分：代码段、数据段、堆、栈
    • 每段独立管理，便于程序设计和调试

  2. 可保护性

    • 每段可设置访问权限（只读、读写、可执行）
    • CPU访问段时检查段界限和权限

  3. 可变长

    • 段大小不固定，按需分配
    • 支持栈和堆动态增长

  4. 地址映射

  • 逻辑地址 → 段表 → 物理地址

  • 如果段内偏移超出段长 → 段界限错误（段越界异常）

分段优缺点

优点：

  1. 逻辑清晰，程序结构映射到内存结构
  2. 可以实现访问控制，提高安全性
  3. 支持段动态增长（堆、栈）

缺点：

  1. 容易产生 外部碎片（段长不固定）
  2. 段表开销大，每个进程需要维护段表
  3. 只分段不能支持虚拟内存，需要结合分页

分页：分成固定大小的块

  • 页号
  • 页内偏移量

**共享页：**多个进程共同使用同一块物理内存页

特点：

  • 节省内存
  • 常用于程序代码和共享库
  • 写入时使用 写时复制（COW）

哈希页表：先用虚拟页号哈希找到槽，再在槽链表里比对虚拟页号，找到对应物理页号，若不存在则缺页

9 虚拟内存管理

优点：

  • 程序数据在物理内存中是分散的，在虚拟内存中是独立连续的
  • 实现逻辑的扩容
  • 地址空间保护

请求调页

**无效位：**这块数据目前不在物理内存里，还在硬盘上

**有效位：**这页数据已经在物理内存里了

缺页错误的处理过程

  • CPU 访问逻辑地址，查页表发现 状态位为“无效”（Invalid），立即触发一个“缺页异常”中断
  • 如果地址合法，OS 在物理内存中找一个空闲的页框
  • OS 启动磁盘调度，将需要的页从磁盘调入刚才找好的物理内存块中
  • 将页表中的“无效”改为“有效”，填入物理块号。最后，重新执行刚才被中断的那条指令。

从页表中找页号发现是无效位，操作系统接管，从磁盘中加载需要的页到物理内存的空闲块中，将页表的无效位修改为有效位
