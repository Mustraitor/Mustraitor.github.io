计算机组成原理

计组

计算机体系结构

  • CPU：实际执行程序的部分
  • 微处理器：单个硅片上上实现的CPU
  • 寄存器：CPU内部用来存放数据的存储单元
  • 时钟：用以生成连续的间隔固定的电脉冲流
  • 指令集体系结构（ISA）： CPU 对程序员（或编译器）提供的“接口规范（程序员可见的）

程序的执行过程

指令集体系结构（ISA）

  • 精简指令集计算机（RISC）：ARM架构

  • 复杂指令集计算机（CISC）：x86架构（Intel）

指令格式

  • 操作码
  • 操作数（operand）

计算机指令执行流程：

  1. 读取指令：CPU 根据地址从内存提取指令。
  2. 解码指令：分析操作码确定操作类型。
  3. 获取数据：根据操作数地址取数。
  4. 执行运算：在加法器等运算器中处理。
  5. 存储结果：将结果存回存储单元。

指令处理数据的方式

  • 存储器—存储器型：它直接从内存中读取数据，原地处理完（比如做个加法），然后直接写回内存。
  • 寄存器—存储器型：一条指令可以处理两个数据，其中一个在内存里，另一个在 CPU 内部的寄存器里。结果可以存回内存或寄存器。
  • 寄存器—寄存器型：只能对寄存器的内容进行操作，Load/Store 型架构，如ARM和MIPS CPU体系架构

存储层次

  • 寄存器: 存放处理器的工作数据
  • Cache: 缓存常用数据的快速存储器
  • DRAM: 存放工作数据
  • 硬盘: 保存程序和数据

总线：连接计算机功能部分，用于数据传输

  • 位宽：总线能够同时传输的并行数据通路的数量
  • 带宽：衡量信息通过总线传输速率的指标，通常用“字节/秒”或“位/秒”表示。
  • 延迟：等待时间，从发出数据传输请求到实际开始传输之间。

存储程序计算机

存储程序计算机结构

解释：

  • 高速暂存寄存器（scratchpad register）
  • 存储器地址寄存器（MAR）：保存了读或写操作正在访问的存储单元的地址
  • 存储器数据寄存器（MBR）：保存了读或写操作正在访问的存储器的数据
  • 指令寄存器（IR）：存放当前正在执行的指令
  • 程序计数器（program counter）：包含下一条将要被执行的指令的地址
  • 循环计数器（Loop Counter）: 用来记录循环执行的次数
  • 寄存器文件（r0-r7）：存放临时文件

三种指令格式

寄存器的可见性

  • 通用寄存器（r0 r1…r15）:存放临时数据
  • 特殊功能寄存器
  • 不可见寄存器

流控制

条件码寄存器（CCR）

四个标志位

  • Z (Zero) 零标志：如果运算结果等于 $0$，则 $Z=1$；否则 $Z=0$。
  • N (Negative) 负标志：如果运算结果是负数（最高位为 $1$），则 $N=1$。
  • V (Overflow) 溢出标志：如果运算超出了寄存器能表示的范围（比如两个正数相加变成负数），则 $V=1$。
  • C (Carry) 进位标志：如果运算产生了进位（加法）或借位（减法），则 $C=1$。

ISA的组成

  • 寄存器集
  • 寻址方式
  • 指令格式

寄存器编码

  • 实现片上存储
  • 访问便捷性和相应时间

形式地址 ($A$)：指令字中直接给出的地址码。

有效地址 ($EA$)：操作数在内存中的真实物理地址。

寻址方式

  • 立即数寻址

  • 直接寻址 $EA = A$

  • 间接寻址 $EA = (A)$

按字节编址：是指内存中的每一个字节（Byte，即 8 位）都有一个唯一的地址编号。

按字编址：每个地址编号对应 1 个字 (Word)。

数据对齐

大小端（Endianness）决定了这些字节在内存地址里的排列顺序

  • 大端方式 (Big-Endian)：把数据的高位字节存放在内存的低地址端

  • 小端方式 (Little-Endian)：把数据的低位字节存放在内存的低地址端

Cache

Cache设计

规则

  • 主存块的大小与cache 的数据区的块一致
  • 一个cache块只能存一个主存块

主存映射方式

  • 直接映射：只能映射到 唯一一个 Cache 块
  • MM 64B, block size 4B

tags 标记位，映射到cache的行号，offset块内地址

  • 全相联映射：可以映射到 任意一个Cache 块
  • 组相联映射：只能映射到 某一组中的任意一个Cache 块

未命中类型

  • Compulsory Miss
  • Conflict Miss( Collision, Interference)
  • Capacity Miss

缓存一致性

共享资源数据存储在多个缓存中

问题：更新其中一个缓存中的共享数据，保证所有缓存数据的一致性

术语：

  • modified
  • shared
  • invalid
  • exclusive
  • owned
  • forward

单个cache

写命中

  • write through: 把所有缓存中和主存中的数据修改
  • write back: 修改缓存块数据的值，替换时才更新主存（一位脏位）

写不命中

  • write allocate: 物理地址对应主存块被 加载 到 cache 块中
  • write unallocate:不加载 主存块至 cache 中，直接更新主存块，只有当执行读操作时才将主存块加载进入 cache 块

缓存一致性协议(多个cache之间)

  • snooping-based protocol
  • directory-based protocol

替换算法

LRU 最近最少使用：替换这组里最久没被访问的那一个块。

FIFO 先进先出：最早进入缓存的那一个最先被替换。

主存储器

  • RAM（SRAM, DRAM）
  • ROM

随机存储器（RAM）：一种使用随机存取方式的存储器设备

随机存取：任意位置的数据都能在相同时间快速访问（RAM ROM都可以）

DRAM 读取是破坏性读取

DRAM刷新：定期重新给电容充电，否则 1 会变成 0。（按行读取）

刷新方式：

  • 集中刷新（Burst Refresh），暂停数据访问，在短时间内快速 刷新所有行，效率高但延迟大；
  • 分散刷新（Distributed Refresh）：每隔一定时间刷新一行。
  • 异步刷新（Asynchronous Refresh），由外部控制器根据需要触发刷新，灵活但依赖控制器设计

字 (Word) ：指 CPU 一次性能够处理的数据长度

位模式——信息表示

  • 指令
  • 数量
  • 字符

无符号整数的机器码是原码

补码（Two’s Complement） 是计算机中表示 有符号整数 的方法。

0为正数，1为负数

正数补码 = 原码

负数补码 = 二进制反码 + 1

n 位补码表示范围：$[-2^{n-1}, 2^{n-1}-1]$
优点：

  • 只有一个0
  • 加减法运算统一
  • 符号位可以参与运算
  • 运算规则简单

易错点：

  • 减法必须变加法
  • 最高位进位丢弃
  • 最小数没有相反数

原码特点：

  • 符号位独立于数值位
  • 正负数的数值位相同
  • 运算复杂（需要判断符号）
  • 存在+0和-0两个零

IEEE浮点数

浮点数的组成

规格化的情况：

一个非0的IEEE 754浮点数可被定义为：

  • X为真值
  • 符号位（ S）：1位
  • 阶码/偏置指数（E）：8位
  • 偏置量（B）：单精度127，双精度1023
  • 尾数的小数部分（F）：23位

非规格化：

  • E = 0时，出现非规格化

E = 0 和 E = 255特例分别用于表示浮点0、非规格化小数、正或负无穷大、NaN

IEEE 754 单精度浮点数转换

  1. 定符号 ($S$)：正数 $S=0$，负数 $S=1$。
  2. 转二进制并规格化：将绝对值转为二进制，移动小数点写成 $1.F \times 2^e$ 的形式，其中 $e$ 是真实指数。
  3. 算阶码 ($E$)：用真实指数 $e + 127$（偏置值），结果转为 8 位二进制。
  4. 填尾数 ($M$)：取 $1.F$ 小数点后的 $F$ 部分，在右边补 $0$ 凑足 23 位。
  5. 拼凑并转换：按 $S(1位) + E(8位) + M(23位)$ 顺序拼接，每 4 位一组转为十六进制。

类型转换

  • 显式类型转换
  • 隐式类型转换：低精度类型会被提升到高精度类型

符号位扩展

  • 如果是负数，前面补 1。
  • 如果是正数，前面补 0。

截断

  • 高位 => 低位
