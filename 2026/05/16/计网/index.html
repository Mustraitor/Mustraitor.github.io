应用层

应用层协议应当定义：

  • 应用进程的报文协议
  • 各种报文类型的语法
  • 字段的语义
  • 进程发送报文的时机与方式，报文响应规则

域名系统DNS

DNS使用传输层的UDP，不要求可靠传输

域名服务器：根域名 => 顶级域名 => 权限域名

域名解析成IP地址的方式：迭代查询，递归查询

DNS 查询次数

  • 0次：没有发出 DNS 查询，本地缓存命中
  • 3次：比如 www.chatgpt.com (根（.）、顶级(.com)、权限(chatgpt.com)、域名服务器)

DNS类比成文件夹

文件传送协议 FTP

控制连接：端口 21 （持久连接）

数据连接：端口 20 或随机端口（取决于主/被动模式）（非持久连接）

FTP 的完整工作流程

  1. 建立控制连接
  2. 发起数据传输方式
  3. 建立数据连接
  4. 传输数据
  5. 退出

超文本传送协议 HTTP

非持久 HTTP

  1. 开启 TCP 连接。

  2. 通过该 TCP 连接最多发送一个对象（如一个 HTML 文件或一张图片）。

  3. 关闭 TCP 连接。

  4. 下载多个对象需要建立多个连接。

持久 HTTP

  1. 向服务器开启 TCP 连接。
  2. 在客户端与该服务器之间的单个 TCP 连接上，可以发送多个对象。
  3. 通信结束后才关闭 TCP 连接。

持久 HTTP的两种工作方式

  • 非流水线方式
  • 流水线方式

传输层

  1. 故事背景：两家人的通信
  • Ann 的家里有 12 个孩子，Bill 的家里也有 12 个孩子。
  • Ann 家的孩子经常给 Bill 家的孩子写信。
  • Ann 和 Bill 是家长，负责收集自家孩子的信并交给邮递员，或者从邮递员那里拿回信件并分发给家里的某个孩子。
故事中的角色 计算机网络概念 说明
房子 (Houses) 主机 (Hosts) 比如你的电脑或手机。
孩子 (Kids) 进程 (Processes) 运行在电脑上的程序（如浏览器、微信）。
信件 (Letters) 应用层消息 (App Messages) 实际要传输的数据。
Ann 和 Bill 传输层协议 (UDP/TCP) 负责进程间的通信。Ann 只管把信送到邮局，并不负责走路。
邮政服务 (Postal Service) 网络层协议 (IP) 负责主机间的通信。邮局只管把信从一栋房子送到另一栋房子。

**TCP：**可靠按序传输、拥塞控制、流量控制、连接建立

使用IP地址&port 端口号，可以唯一确定一个进程

套接字 = IP地址 + port 端口号

**UDP：**不可靠、无序交付、速度快

UDP不可靠传输

UDP差错检验：通过伪首部，计算检验和

  • 回卷：进位加回到尾部

  • 取反码

**传输无误：**全 1

**传输有差错：**出现 0

都无法保证延迟、带宽

TCP可靠传输的工作原理

  1. 分段
  2. 编号 + 顺序重排 （序列号）
  3. ACK确认 （ACK号）
  4. 超时重传 （定时器）
  5. 滑动窗口

停止等待协议：针对一个数据的可靠传输

**连续ARQ协议：**多个数据同时传输，无需等待确认（流水线传输）

​ 对于流水线错误恢复，可以采用两种基本方法：回退 N 步（Go-Back-N, GBN） 和 选择重传（Selective Repeat, SR）。

拥塞控制

防止过多数据注入到网络中导致拥塞

拥塞控制方法

MSS 最大报文段长度：每一个TCP报文段中的数据字段的最大长度（仅包含应用层数据）

比如 2000B 的数据，假设1 MSS = 500 B ，则需要 4 个报文段

**接收窗口（$W_r = 1$）：** 接收方**只能按序接收**分组。如果收到的分组序号不是它期望的下一个，它会直接丢弃，并对最近一个按序到达的分组重新发送确认（ACK）。 - **累积确认（Cumulative ACK）：** 如果接收方发送了 `ACK n`，代表序号为 $n$ 及其之前的所有分组都已经正确收到了。 ### 选择重传协议（SR） - **发送窗口 ($W_s > 1$)**：允许发送方发送多个分组，即使没收到确认。 - **接收窗口 ($W_r > 1$)**：这是与 GBN 最大的不同！**接收方也有窗口且大于 1**，这意味着它可以**缓存乱序到达的分组**。 - **独立确认**：接收方对每一个收到的正确分组发送**单独的确认（ACK）**，而不是累积确认。 - $W_s + W_r \le 2^k$ ， $ W_r \le W_s$ 例子：窗口大小为 7 ,就是包含7个序号[0 1 2 3 4 5 6] ### 信道利用率计算公式 $$U = \frac{W_st_t}{t_t + RTT + t_a}

CRC计算过程

CRC 的核心目标：是让发送的比特流 <D,R><D, R><D,R> 能够被双方约定的生成多项式 GGG 整除。

DDD (Data)：待发送的原始数据比特流。

GGG (Generator)：生成多项式（除数），长度为 r+1r+1r+1 位，由协议预先约定。

rrr：校验位的长度，通常比 GGG 的位数少一位。

RRR (CRC bits)：计算出的校验码（余数），长度为 rrr 位。（冗余码FCS帧检验序列）

角色 步骤 操作说明
发送方 1. 补零移位 将 DDD 左移 rrr 位（即 D⋅2rD \cdot 2^rD⋅2r），末尾补 rrr 个 0。
2. 模 2 除法 用补零后的数据除以 GGG，求出 余数 RRR。
3. 拼接发送 将 RRR 替换掉末尾的 0，发送数据 <D,R><D, R><D,R>。
接收方 1. 校验除法 用收到的数据直接除以约定的 GGG。
2. 结果判定 若余数为 0，则数据正确；若非 0，则检测到错误。

如果当前被除数的最高位是 1，商就是 1，然后用除数 GGG 去异或被除数。

如果当前被除数的最高位是 0，商就是 0，然后用全 0 去异或被除数。

通信知识

码元：代表不同离散数值的基本波形

全双工：通信双方可以同时发送和接受信息

半双工：通信双方不可以同时发送和接受信息，但都可以发送和接收信息

曼切斯特编码：位周期中心的跳动

奈氏准则：在带宽为W（Hz）的低通信道中，若不考虑噪声影响，则码元传输的最高速率是2W （码元/秒）

信噪比：信号的平均功率与噪声的平均功率之比 S/N

香农公式： C=Wlog⁡2(1+SN)C = W\log_2 (1 + \frac{S}{N})C=Wlog2​(1+NS​)

W: 信道的带宽

C：信道的极限信息传输速率 bit/s
